## 🎯 لماذا هذه المهارة مهمة جداً؟

التوكنز هي "وحدة قياس" استهلاك الذاكرة والتكلفة في نماذج الذكاء الاصطناعي.
كلما قلّ استهلاك التوكنز، كان النظام:
- **أسرع بـ 10 أضعاف** (سياق أقل = معالجة أسرع)
- **أرخص تكلفةً** (بخاصة في جلسات العمل الطويلة)
- **لا ينسى التعليمات المهمة** (منع امتلاء نافذة السياق Context Window)
- **أكثر تركيزاً ودقة** (بيانات أقل = قرارات أوضح)

## 📋 النظام الرباعي للبروتوكولات

### البروتوكول A: سير العمل الثلاثي الإلزامي (Claude-Mem)
```
الخطوة 1 → search(query)           ← ~100 توكن، يُعيد أرقام تعريفية فقط
الخطوة 2 → timeline(anchor=ID)     ← ~300 توكن، يُعيد السياق المحيط
الخطوة 3 → get_observations([IDs]) ← ~1,500 توكن، 3 إلى 5 نتائج فقط
```
**قانون صارم:** لا يجوز أبداً الاستدعاء المباشر لـ `get_observations` على كل النتائج.

### البروتوكول B: الملاحة الذكية في الأكواد
```
❌ خطأ:  view_file("script.py")        ← يقرأ 800 سطر بشكل أعمى
✅ صح:   grep_search("def ", file.py)  ← يجلب أسماء الدوال فقط
✅ صح:   smart_outline(file.py)        ← هيكل AST مطوي
✅ صح:   smart_unfold(function_name)   ← دالة واحدة محددة
```

### البروتوكول C: التعديل الجراحي الدقيق
```
❌ خطأ: إعادة كتابة الملف كاملاً
✅ صح:  multi_replace_file_content مع TargetContent المحدد
✅ صح:  replace_file_content لكتلة متصلة واحدة فقط
```

### البروتوكول D: الكوربس للملفات الضخمة
```
للملفات التي تتجاوز 500 سطر أو 20,000 توكن:
  1. build_corpus(name, query)    ← فهرسة المحتوى
  2. prime_corpus(name)           ← إنشاء جلسة ذكاء اصطناعي مركّزة
  3. query_corpus(name, question) ← أسئلة ومعالجة دقيقة
```
هذا يمنع إغراق الذاكرة النشطة بمستندات ضخمة.

## ⚠️ الأنماط المحظورة تماماً

| الفعل المحظور | السبب | البديل |
|--------------|-------|--------|
| `view_file` على ملف > 100 سطر | يهدر 1,000–5,000 توكن | `grep_search` + `smart_unfold` |
| `get_observations` بدون `search` مسبق | يجلب بيانات غير ذات صلة | استخدم سير العمل الثلاثي |
| `write_to_file(Overwrite=true)` للتعديل | يهدر كل توكنز الإخراج | `multi_replace_file_content` |
| قراءة ملف `.docx` كاملاً للتحقق | مكلف جداً | استخدم أداة فحص مخصصة |
| قراءة كل Knowledge Items في بداية المحادثة | يملأ نافذة السياق | اقرأ فقط ما هو ذو صلة بالمهمة |
